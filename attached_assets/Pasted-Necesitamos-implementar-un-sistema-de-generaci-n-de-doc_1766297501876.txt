Necesitamos implementar un sistema de generación de documentos estructurados (CVs, reportes, cartas) con calidad profesional como el de la imagen; la arquitectura consta de tres capas fundamentales: primero, Structured Output donde el LLM genera JSON estructurado siguiendo un schema predefinido (JSON Schema o Zod) que define la anatomía exacta del documento (para CV: header con nombre/teléfono/email/dirección/website/foto, sections array conteniendo objetos tipados como work_experience con company/role/dates/description/achievements, education con institution/degree/dates, skills con categorías y niveles de proficiencia representados como rating 1-5, languages con idioma y nivel visual tipo dots/progress bar, y profile_summary); segundo, Template Engine usando docxtemplater, carbone.io, o python-docx con plantillas DOCX pre-diseñadas por diseñadores profesionales que contienen placeholders mapeados al schema ({name}, {#work_experience}{company}{/work_experience}), estilos tipográficos definidos (fonts, sizes, colors, spacing), layouts con grids y columnas, y elementos visuales como skill badges, progress indicators, y photo frames con aspect ratio fijo; tercero, Intelligent Mapping Layer donde un módulo intermedio recibe el JSON del LLM, valida contra el schema, transforma datos (fechas a formato legible, skill levels a representación visual de dots/barras), selecciona el template óptimo basado en el tipo de documento o preferencia del usuario, e inyecta los datos en el template generando el DOCX final; implementa además un Prompt Engineering Layer con system prompts específicos por tipo de documento que instruyan al LLM sobre mejores prácticas (para CV: action verbs, métricas cuantificables, keywords de industria), análisis del input del usuario para extraer y estructurar información incluso de texto desordenado, y sugerencias de mejora del contenido; el sistema debe soportar múltiples templates por tipo de documento (moderno, clásico, creativo, minimalista), preview en tiempo real renderizando el template con los datos actuales, edición híbrida donde el usuario puede modificar tanto el JSON estructurado como el documento visual, y generación de foto profesional con IA (stable diffusion con img2img) o integración para upload de foto real con auto-crop y filtros; expón una API unificada que reciba prompt natural del usuario ("crea mi CV, soy desarrollador con 5 años de experiencia en React..."), genere el structured output via LLM, aplique el template seleccionado, y retorne el documento final en DOCX/PDF con preview HTML para el editor visual.
