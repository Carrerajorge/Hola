Implementa un módulo ANALYTICS ENGINE enterprise-grade con arquitectura event-driven: 1) REAL-TIME DASHBOARD - WebSocket connection (Socket.io) para streaming de métricas live sin polling, cards KPI con animación de counter-up mostrando: active_users_now, queries_per_minute, tokens_consumed_today, revenue_today, avg_latency_ms, error_rate_percentage. 2) CHARTS INTERACTIVOS - usar Recharts con granularidad selector (1h/24h/7d/30d/90d/1y), incluir: UserGrowthChart (area), RevenueTrendChart (line con gradient), ModelUsageChart (stacked bar), LatencyByProviderChart (multi-line comparativo para OpenAI/Anthropic/xAI/DeepSeek/Google), ErrorRateChart (área con threshold line), TokenConsumptionChart (bar por modelo). 3) TRACKING SYSTEM - capturar eventos: session_start, session_end, query_sent, response_received, error_occurred, payment_completed, plan_upgraded, plan_downgraded; cada evento incluye: event_id mod (UUID), user_id, timestamp, metadata{}. 4) PERFORMANCE METRICS - tabla de latencia por provider con columns: provider, avg_latency, p50, p95, p99, success_rate, total_requests, status_indicator (green/yellow/red basado en thresholds). 5) COST TRACKING - panel de costos real-time por provider con: current_spend, budget_limit, percentage_used, projected_monthly, alert_threshold; implementar AlertBanner cuando spend > 80% budget. 6) FUNNEL ANALYSIS - visualización de conversión: visitors → signups → free_active → trial_started → pro_subscribed → enterprise_upgraded, con porcentajes entre cada step y comparison vs previous_period. 7) COHORT RETENTION - heatmap table mostrando retención por semana de signup (Week 0 a Week 12), color-coded de verde (high) a rojo (low). 8) USAGE HEATMAP - grid 24x7 (horas x días) con intensidad de color basada en query_volume, tooltip con exact_count. 9) API LOGS EXPLORER - tabla paginada con: timestamp, user, endpoint, method, status_code, latency, tokens_in, tokens_out, model, request_preview (truncado), response_preview; filtros por: date_range, user, status, model, latency_range; búsqueda fulltext en request/response. 10) EXPORT SYSTEM - botones para generar CSV (client-side con Papa Parse) y PDF (con jsPDF + autoTable) de cualquier vista/tabla con date_range seleccionado. 11) ALERTS CONFIG - UI para crear alertas: metric_selector, operator (>, <, =), threshold_value, webhook_url, notification_channels (email/slack/webhook); mostrar active_alerts list con last_triggered timestamp. 12) DATA STRUCTURE - events[] con {id, type, user_id, timestamp, data{}}, metrics_snapshots[] con {timestamp, active_users, queries, tokens, revenue, latency_avg, error_count}, provider_stats[] con {provider, requests, tokens, cost, avg_latency, error_rate}. 13) STATE MANAGEMENT - usar React Context con useReducer para analytics state global, implementar custom hooks: useRealtimeMetrics(), useAnalyticsChart(metric, granularity), useEventStream(). 14) UI/UX - diseño dark mode consistente, skeleton loaders durante data fetch, números con formatting (1.2K, $45.6K), relative timestamps, tooltips informativos, responsive grid layout que adapta charts en mobile. Priorizar performance con React.memo, useMemo para cálculos pesados, y virtualización para tablas largas.
