Un “GPT” en ChatGPT (los “custom GPTs”) no es un modelo nuevo entrenado desde cero, sino una configuración/“envoltura” sobre el mismo modelo base de ChatGPT (que se migra automáticamente al equivalente más cercano del modelo disponible) donde el builder define: (1) instrucciones (reglas de comportamiento y objetivo), (2) conocimiento (archivos que se adjuntan como base documental), y (3) capacidades/herramientas (p. ej., navegación web, análisis de datos, generación de imágenes y/o Actions para llamar APIs externas); externamente, para el usuario se ve como “otro bot” especializado, pero internamente la arquitectura típica es una tubería tipo orquestación + LLM + herramientas: cuando envías un mensaje, el sistema construye un prompt compuesto con (a) políticas y system instructions de la plataforma, (b) las instrucciones del GPT (lo que configuró el builder), (c) el historial relevante de la conversación, y (d) contexto dinámico (por ejemplo, retrieval desde los archivos de “Knowledge”); luego el modelo hace inferencia para producir texto y/o decidir si debe usar una herramienta: si activaste “Knowledge”, el runtime aplica técnicas de búsqueda semántica / RAG para seleccionar fragmentos pertinentes de los archivos (en el editor se pueden adjuntar hasta 20 archivos) y los inyecta como contexto para que el modelo responda con más precisión; si activaste GPT Actions, el modelo puede decidir emitir una llamada estructurada a una API REST descrita (típicamente mediante un esquema/OpenAPI), el sistema ejecuta esa llamada fuera del modelo, recupera la respuesta de la API y se la vuelve a pasar al modelo para que redacte el resultado final en lenguaje natural; todo esto ocurre con controles de seguridad y permisos (p. ej., qué herramientas están habilitadas, qué puede invocar el GPT y bajo qué condiciones), y en el lado de construcción, el GPT Builder que ves en la UI es, “por debajo”, otro custom GPT con instrucciones y una acción que escribe/actualiza los campos de configuración del GPT que estás creando, lo que explica por qué “conversar” con el builder termina en cambios concretos de nombre, descripción, instrucciones y capacidades del GPT.
