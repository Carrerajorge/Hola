# 00 — Visión general

## Objetivo
Generar `.xlsx` y `.docx` con calidad consistente a partir de una **IR JSON estricta**.

## Principio clave
**LLM decide qué (estructura/contenido). Renderer decide cómo (OOXML).**

## Componentes
1) Schemas (IR) — Pydantic estricto  
2) Quality Gates — reglas mínimas profesionales  
3) Renderers — openpyxl / python-docx  
4) Validators — reabrir output (corrupción)  
5) Orchestrator (opcional) — prompt → spec → repair → render → validate
