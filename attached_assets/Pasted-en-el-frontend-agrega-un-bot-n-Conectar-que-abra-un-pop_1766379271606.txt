en el frontend agrega un botón Conectar que abra un popup OAuth (Google Identity Services) hacia tu backend (GET /auth/google/start) usando Authorization Code + PKCE, con scope mínimo necesario (p.ej. openid email profile https://www.googleapis.com/auth/forms.body https://www.googleapis.com/auth/drive.readonly), access_type=offline y prompt=consent para obtener refresh_token; el backend completa el callback (GET /auth/google/callback) validando state, intercambiando el code por tokens y guardando cifrado {userId, provider:"google", access_token, refresh_token, expiry}; al volver del popup, el frontend llama POST /mcp/sessions para que tu backend cree una sesión MCP (por usuario) conectándose a tu MCP Server con transporte Streamable HTTP o SSE (si sigues tu ejemplo FastMCP, expón server.run(transport="sse", host, port) y publica una URL tipo https://api.tu-dominio.com/mcp/sse) y en tu “MCP client” (SDK oficial o FastMCP Client) inicializa el transporte HTTP/SSE apuntando a esa URL y adjunta Authorization: Bearer <token_app> o un mcp-session-id propio para mapear sesión↔usuario (en MCP la autorización va a nivel de transporte y se basa en OAuth 2.x cuando aplique); luego el backend registra/descubre herramientas del servidor (p.ej. search, fetch, y tus tools forms.list, forms.get, forms.create, forms.update) y, cuando el usuario interactúe en el editor, tu app invoca esas tools vía MCP pasando el contexto y usando el access_token de Google (renovándolo con refresh_token antes de expirar) para que la tool realmente lea/escriba en Google Forms; finalmente, implementa revocación (POST /auth/google/disconnect) borrando tokens, cerrando sesión MCP, y audita seguridad (PKCE, state, sameSite cookies o JWT, CORS estricto, cifrado en reposo, rotación de claves).
